Workflow #2

Application Iteration

Change specifications Flow to Prototypes. Promote squash-merges To Projects.

flowchart LR classDef dir fill:#0a5c38,stroke:#2cb67d,color:#fff,font-weight:bold classDef md fill:#d4a017,stroke:#a07810,color:#111,font-weight:bold classDef script fill:#1e40af,stroke:#3b5fc0,color:#fff TI{{"Commit Information"}}:::md --> S1["iterate.sh"]:::script CH{{"Specification Diff"}}:::md --> S1["iterate.sh"]:::script PT1(["Prototype/"]):::dir --> S1 S1 --> PT2(["Prototype/"]):::dir S1 --> SC2{{"SCORECARD.md"}}:::md PT2 --> S2["merge.sh"]:::script SC2 --> S2 S2 --> PROJ(["Project"]):::dir
Learnings

I much prefer specification driven updates. Tickets have numeric NNN in the name so they can be applied in order. Acceptance Criteria tickets keep specification drift minimal.

Inputs

  • TICKETS.md -- By Type (screen,feature...), Acceptance Criteria
  • Prototype/

Outputs

  • Prototype/ (updated)
  • SCORECARD.md (updated)
  • Project (squash-merged)
Workflow #3

Technology Rules Propagation

Technology interfaces are injected into projects using CLAUDE_RULES.md - an AI summarization from application perspective of Technology Rules (not Stack Build Rules). This guidance enables guidance other programs to call/control application behaviors and transparently lets the application interface with the technology ecosystem

Learnings

CLAUDE_RULES injection worked really well out of the box. Key insight: use a a crafted ai summary

An opinionated prescribed stack gave me working software first time

Inputs

  • BUSINESS_RULES.md
  • templates/
  • stack/

Outputs

  • CLAUDE_RULES.md
  • Project AGENTS.md (injected)
  • PROJECT_SCORECARD.md
Workflow #4

Specification Review

AI-scored gap analysis of the specification directory — produces a prioritized gap list (P0–P4), quality scorecard, and next iteration prompt.

flowchart LR classDef dir fill:#0a5c38,stroke:#2cb67d,color:#fff,font-weight:bold classDef md fill:#d4a017,stroke:#a07810,color:#111,font-weight:bold classDef script fill:#1e40af,stroke:#3b5fc0,color:#fff GAPS{{"REFERENCE_GAPS.md"}}:::md --> SI["spec_iterate.sh"]:::script SPEC(["Specifications/"]):::dir --> SI SI --> SCORE{{"SPEC_SCORECARD.md"}}:::md SI --> ITER{{"SPEC_ITERATION.md"}}:::md
Learnings

Lorem ipsum dolor sit amet, asking the AI to review the specification before the build revealed blind spots a human reviewer would miss consistently — missing error states, unspecified edge cases, ambiguous field names.

Inputs

  • Specifications/
  • BUSINESS_RULES.md

Outputs

  • REFERENCE_GAPS.md
  • SPEC_SCORECARD.md
  • SPEC_ITERATION.md
