Workflow

Each step has one command. Edit spec files between steps.

1 bin/create_spec.sh MyProject "description" — Scaffold spec directory from templates
2 bin/validate.sh MyProject — Check required files, naming, fields
3 bin/convert.sh MyProject > prompt.md — Generate expansion prompt for AI
4 bin/build.sh MyProject > prompt.md — Tag commit and generate build prompt
5 bin/test.sh — Run unit tests on the specification system
Iterate: edit specs, re-validate, re-build. Each build creates an annotated git tag recording the spec state.
