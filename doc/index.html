Specifications

Write concise specs. Expand with AI. Build in one shot.

Workflow

Each step has one command. Edit spec files between steps.

1 bin/create_spec.sh MyProject "description" — Scaffold spec directory from templates
2 bin/validate.sh MyProject — Check required files, naming, fields
3 bin/convert.sh MyProject > prompt.md — Generate expansion prompt for AI
4 bin/build.sh MyProject > prompt.md — Tag commit and generate build prompt
5 bin/test.sh — Run unit tests on the specification system
Iterate: edit specs, re-validate, re-build. Each build creates an annotated git tag recording the spec state.

File Architecture

bin/create_spec.sh generates all files from templates. Spec files end with ## Open Questions (except README, METADATA, INTENT).

METADATA.md
Project identity: name, status, description
README.md
One-line project description
INTENT.md
Why this project exists, scope boundaries
ARCHITECTURE.md
Modules, routes, directory layout
DATABASE.md
Tables and columns — delete if no database
UI.md
Shared UI patterns — delete if no UI
SCREEN-{Name}.md
Per-screen: route, layout, interactions
FEATURE-{Name}.md
Cross-cutting behavior: trigger, sequence
