# 04 — Lambda Function & Alexa Skill

The Lambda function is the cloud-side entry point. It receives every Alexa intent,
validates the request, extracts the utterance, enqueues it to SQS, and returns an
Alexa confirmation response. All routing logic happens in the local daemon — Lambda
never calls localhost.

---

## SQS Message Envelope (Canonical Definition)

This is the contract between Lambda (producer) and the daemon (consumer).
Both sides must use exactly this schema. Do not add fields without updating both.

```json
{
  "message_id": "uuid4-string",
  "keyword_raw": "full utterance text from Alexa, lowercase normalized",
  "alexa_session_id": "amzn1.echo-api.session.xxx",
  "user_id": "amzn1.ask.account.xxx",
  "device_id": "amzn1.ask.device.xxx",
  "timestamp": "2026-03-11T14:23:01Z"
}
```

- `message_id`: generated by Lambda via `str(uuid.uuid4())`
- `keyword_raw`: `slots['utterance']['value']` lowercased and stripped
- `timestamp`: UTC ISO-8601, `datetime.utcnow().strftime('%Y-%m-%dT%H:%M:%SZ')`
- `alexa_session_id`, `user_id`, `device_id`: extracted from Alexa request JSON

## Alexa Skill Configuration

**Skill type:** Custom
**Invocation name:** "command center" (user says "Alexa, open command center")
**Languages:** en-US

### Interaction Model

**One intent: `VoiceCommandIntent`**

Using a single free-text intent is the correct architecture. Adding separate
intents per keyword (CommandIntent, IdeaIntent, etc.) would require Alexa model
redeployment every time routing rules change. With a single free-text intent,
routing is controlled entirely via the `routing_rules` SQLite table.

```json
{
  "name": "VoiceCommandIntent",
  "slots": [
    {
      "name": "utterance",
      "type": "AMAZON.SearchQuery"
    }
  ],
  "samples": [
    "{utterance}",
    "tell me {utterance}",
    "command {utterance}",
    "note {utterance}"
  ]
}
```

**Use `AMAZON.SearchQuery`, not `AMAZON.AlphaNumeric`.**
`AMAZON.SearchQuery` is designed for free-form natural language phrases and
captures multi-word utterances reliably. `AMAZON.AlphaNumeric` truncates at
word boundaries and misses common words. Test with utterances 5+ words long.

Also include standard built-in intents (required for certification):
- `AMAZON.StopIntent`
- `AMAZON.CancelIntent`
- `AMAZON.HelpIntent`

### Endpoint

Set endpoint type to **Lambda function ARN** (not HTTPS).
Paste the Lambda function ARN from the AWS console after deploying.

## Lambda Function: `lambda_function.py`

```python
import json
import os
import uuid
import logging
from datetime import datetime, timezone

import boto3

logger = logging.getLogger()
logger.setLevel(logging.INFO)

sqs = boto3.client('sqs', region_name=os.environ['AWS_REGION'])
QUEUE_URL = os.environ['SQS_QUEUE_URL']
SKILL_ID = os.environ['ALEXA_SKILL_ID']


def lambda_handler(event, context):
    """Entry point for all Alexa requests."""
    # 1. Validate skill ID
    application_id = event.get('session', {}).get('application', {}).get('applicationId', '')
    if application_id != SKILL_ID:
        logger.warning(f"Rejected request from unknown skill: {application_id}")
        return _error_response("Unauthorized skill.")

    request_type = event.get('request', {}).get('type', '')

    # 2. Handle session lifecycle
    if request_type == 'LaunchRequest':
        return _speech_response("Command center ready. Go ahead.", end_session=False)

    if request_type == 'SessionEndedRequest':
        return _empty_response()

    if request_type != 'IntentRequest':
        return _empty_response()

    intent_name = event['request']['intent']['name']

    # 3. Handle built-in intents
    if intent_name in ('AMAZON.StopIntent', 'AMAZON.CancelIntent'):
        return _speech_response("Goodbye.", end_session=True)

    if intent_name == 'AMAZON.HelpIntent':
        return _speech_response("Say anything after opening command center and I'll route it.", end_session=False)

    # 4. Handle VoiceCommandIntent
    if intent_name == 'VoiceCommandIntent':
        return _handle_voice_command(event)

    return _speech_response("I didn't understand that.", end_session=True)


def _handle_voice_command(event):
    slots = event['request']['intent'].get('slots', {})
    utterance_slot = slots.get('utterance', {})
    utterance = utterance_slot.get('value', '').strip().lower()

    if not utterance:
        return _speech_response("I didn't catch that. Try again.", end_session=False)

    session = event.get('session', {})
    user = session.get('user', {})
    context_obj = event.get('context', {})
    system = context_obj.get('System', {})

    message = {
        'message_id': str(uuid.uuid4()),
        'keyword_raw': utterance,
        'alexa_session_id': session.get('sessionId', ''),
        'user_id': user.get('userId', ''),
        'device_id': system.get('device', {}).get('deviceId', ''),
        'timestamp': datetime.now(timezone.utc).strftime('%Y-%m-%dT%H:%M:%SZ'),
    }

    try:
        sqs.send_message(
            QueueUrl=QUEUE_URL,
            MessageBody=json.dumps(message),
            MessageAttributes={
                'source': {'StringValue': 'alexa', 'DataType': 'String'}
            }
        )
        logger.info(f"Enqueued message {message['message_id']}: {utterance[:80]}")
        return _speech_response("Got it.", end_session=True)
    except Exception as e:
        logger.error(f"Failed to enqueue: {e}")
        return _speech_response("Sorry, something went wrong. Try again.", end_session=True)


def _speech_response(text, end_session=True):
    return {
        'version': '1.0',
        'response': {
            'outputSpeech': {'type': 'PlainText', 'text': text},
            'shouldEndSession': end_session,
        }
    }

def _error_response(text):
    return _speech_response(text, end_session=True)

def _empty_response():
    return {'version': '1.0', 'response': {}}
```

## IAM Role

The Lambda function needs an execution role with:

1. **Managed policy**: `AWSLambdaBasicExecutionRole` (CloudWatch Logs)
2. **Inline policy** — SQS send only:

```json
{
  "Version": "2012-10-17",
  "Statement": [{
    "Effect": "Allow",
    "Action": ["sqs:SendMessage"],
    "Resource": "arn:aws:sqs:{region}:{account-id}:alexa-prototype-queue"
  }]
}
```

No other permissions. Lambda cannot read from the queue — that is the daemon's job.

## Configuration

### Lambda Environment Variables

| Variable | Value |
|----------|-------|
| `AWS_REGION` | e.g. `us-east-1` |
| `SQS_QUEUE_URL` | Full queue URL from AWS console |
| `ALEXA_SKILL_ID` | `amzn1.ask.skill.xxxxxxxx-xxxx-...` |

`AWS_ACCESS_KEY_ID` and `AWS_SECRET_ACCESS_KEY` are NOT needed in Lambda —
it uses its IAM execution role automatically.

### Lambda Settings

| Setting | Value |
|---------|-------|
| Runtime | Python 3.12 |
| Handler | `lambda_function.lambda_handler` |
| Timeout | 5 seconds |
| Memory | 128 MB |
| Architecture | x86_64 |

## Deployment

Package and upload as a zip file (no frameworks required for prototype):

```bash
# From lambda/ directory
zip -r lambda.zip lambda_function.py
aws lambda update-function-code \
    --function-name alexa-prototype-handler \
    --zip-file fileb://lambda.zip
```

The Lambda directory is `lambda/` in the project root. It contains only
`lambda_function.py` — boto3 is available in the Lambda Python 3.12 runtime
without packaging.

## Local Testing (Without Alexa)

During development, test the Lambda by sending a crafted event directly:

```bash
aws lambda invoke \
    --function-name alexa-prototype-handler \
    --payload file://test_events/voice_command.json \
    response.json
cat response.json
```

Sample test event at `test_events/voice_command.json`:

{
