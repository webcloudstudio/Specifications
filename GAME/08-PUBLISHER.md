# GitHub Homepage Publisher

`publisher.py` — Generates and publishes a GitHub Pages portfolio site from project card metadata.

## Overview

The publisher takes projects marked with `card_show=1`, renders them as Astro card components, and deploys to a sibling `My_Github` directory that is a separate GitHub Pages repository.

## Directory Structure

```
~/projects/
  commandcenter/          # This app
  My_Github/              # Astro-based GitHub Pages site
    src/pages/index.astro  # Generated by publisher
    ...
```

`PAGES_DIR` is auto-detected as a sibling directory: `../My_Github` relative to commandcenter.

## Three-Step Pipeline

### 1. Rebuild (`rebuild_async`)

Calls `bin/RebuildYourHomepage.sh` via spawn service (batch mode).

The bash script:
1. `cd` to `My_Github` directory
2. `npm run build`

Before the script runs, `rebuild_pages()` generates `index.astro`:
1. Query all projects where `card_show=1`
2. Render each as an Astro `<article>` card via `_render_card()`
3. Wrap in full Astro layout via `_build_astro()`
4. Write to `My_Github/src/pages/index.astro`

### 2. Preview (`start_preview`)

Calls `bin/LocalPreview.sh` via spawn service (daemon mode).

The bash script:
1. `cd` to `My_Github` directory
2. `npm run preview -- --port 4321`

Preview accessible at `http://localhost:4321/webcloudstudio`

### 3. Push (`push_async`)

Calls `bin/PushAndPublish.sh` via spawn service (batch mode).

The bash script:
1. `cd` to `My_Github` directory
2. `git add -A`
3. `git commit -m "Update homepage cards"` (|| true to handle "nothing to commit")
4. `git push`

GitHub Actions handles actual deployment after push.

## Card Rendering (`_render_card`)

Each project card contains:
- **Icon**: Book SVG (📖) or Heart SVG (💻 equivalent)
- **Meta**: Card type label + URL link (domain only)
- **Title**: `card_title` or project title fallback
- **Description**: `card_desc` or project description
- **Tags**: Comma-separated or JSON array, rendered with color classes from `data/tag_colors.json`
- **Action button**: "Live Site ↗" or "View on Amazon ↗" (books) linking to `card_url`

### Tag Colors

Six color categories stored in `data/tag_colors.json`:
```json
{
  "_comment": "Tag color configuration. Colors: py (green), db (orange), ops (blue), fe (purple), sec (red), neutral (gray)",
  "tags": {
    "Python": "py",
    "Flask": "ops",
    "React": "fe"
  }
}
```

CSS class mapping: `tag--py`, `tag--db`, `tag--ops`, `tag--fe`, `tag--sec`, `tag--neutral`

## Astro Layout (`_build_astro`)

Generated `index.astro` contains:
- **Hero section**: Name, title, description
- **Projects section**: Horizontal card carousel with prev/next buttons
- **Carousel JS**: Client-side scroll with responsive breakpoints (1/2/3 cards per view)
- **Inline styles**: Carousel wrapper, nav buttons, scroll container

## Process Tracking

Global variables track current run IDs:
- `_rebuild_run_id`, `_push_run_id`, `_preview_run_id`

Status check functions (`rebuild_is_running()`, etc.) query spawn service for process status.

## Click Time Recording

Each pipeline button click records timestamp to `data/publish_clicks.json`:
```json
{"rebuild": "2024-01-15T10:30:00", "push": "2024-01-15T10:35:00", "preview": "2024-01-15T10:32:00"}
```
Displayed under pipeline buttons as "last clicked" indicator.
