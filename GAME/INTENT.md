## Intent

Important: The INTENT file contains the highest level objectives of the project

**Purpose 1: Best of Breed Developer Experience Dashboard**

Important: The short term target is making a best of breed developer experience for an individual developer or small team

Important: The long term target is "Family and Friends" - a single dashboard enabling you to share components and services with
your family/friends/team.  This can and will include home automation, seamless communication, and easy calendar and messaging.  This platform
will expose these kind of services as a dashboard the user controls and can extend

Manage all your projects on a single Dashboard
    On a single set of dashboards where you have complete observability and control of your projects
    Provides a clear management framework for your projects
        Best for projects you want saved to git or which integrate with other projects
    You generate your application and then integrate with your environment
Projects can be anything you are working over time - and are archived when done.  
Unified view on your projects: status, health, operations, logs, documentation as appropriate.

**Purpose 2: Conform Existing and New Projects to Standards that Enable Purpose 1**

Seamlessly expose the needed services across all your projects and prototypes using minimally invasive approach

Specifically add bash scripts that encapsulate tool functionality in bin/ scritps and then enable a rest api that calls those scripts.  Other languages from Bash can be used if wanted.

Metadata on the project is provided within the project enabling it to expose its capabilities using best practices including 
functionality, logging, command and control, operations, and general/specific endpoints.  This metadata is aggregated in a 
service catalog that ensures unified operations and interoperability.  This platform ensures the UI capabilities.

**Purpose 3: Prototype workflow**

General project workflow enabling you to use Best Design practices trivially within your own methodology.

Provide Look and Feel, Stack Rules, Business Rules so you focus on needed functionality 

**Purpose 4: True Purpose**

Important: This is a Resume builder for the owner.  The project should be best practices and polished and use latest 
look and feel and other items.  Please suggest ways to improve the project look and feel, functionality, or process.
